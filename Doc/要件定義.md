# PhotoDiary（アルバム管理）設計書

---

# 1. システム概要

撮影した写真を一元管理し、日記形式で閲覧できるアルバムアプリ。
ユーザーごとに写真・グループを管理し、日付・場所・カテゴリで整理・検索可能。
さらに、複数写真の選択による一括削除や、個別写真の編集・削除機能を追加。

---

# 2. システム構成

* フロントエンド：React + TypeScript
* バックエンド：Spring Boot
* DB：MySQL
* ストレージ：ファイルサーバ or クラウド（S3想定）

---

# 3. 写真保存方式

## ■ 方針

### パターン①（推奨）

* 画像ファイル：サーバ or S3 に保存
* DB：パスのみ保持

```
例）/images/{userId}/{yyyy}/{MM}/{uuid}.jpg
```

### 理由

* DB肥大化防止
* パフォーマンス向上
* CDN利用可能

# 5. 画面設計

## 5.1 ログイン画面

```
+----------------------+
|   PhotoDiary         |
|----------------------|
| ユーザー名 [      ] |
| パスワード [      ] |
|                      |

```

---

## 5.2 メイン画面（アルバム一覧）

```
+--------------------------------------------------------------+
| 年[▼] 月[▼] 日[▼] グループ[検索____] [絞込] [ログアウト]     |
+----------------------+---------------------------------------+
| 2026                 | □写真1                                |
|  └ 04                | □写真2                                |
|     └ 01            | □写真3                                |
|     └ 02            | □写真4                                |
|                      |                                       |
| 2025                 | □写真5                                |
+----------------------+---------------------------------------+
```

### ■ ポイント

* 左：日付ツリー
* 右：写真一覧
* スクロール連動
* 写真にチェックボックスを追加して、複数選択可能
* 選択写真に対して「編集」「削除」「一括削除」操作が可能
* 写真の一括登録が可能（写真をマルチ選択して登録できる）
* 写真の場所・説明を一括更新できる（写真をマルチ選択して編集できる）

---

# 6. API設計（概要）

| メソッド   | URL              | 内容               |
| ------ | ---------------- | ---------------- |
| POST   | /api/login       | ログイン             |
| GET    | /api/photos      | 写真一覧取得           |
| POST   | /api/photos      | 写真登録             |
| PUT    | /api/photos/{id} | 写真編集             |
| DELETE | /api/photos/{id} | 写真削除（単体）         |
| DELETE | /api/photos      | 写真削除（一括、ID配列で送信） |
| POST   | /api/photos/bulk | 写真一括登録（複数ファイル同時アップロード） |
| PUT    | /api/photos/bulk | 写真一括更新（場所・説明をID配列で一括変更） |
| GET    | /api/photo-groups      | グループ一覧           |
| POST   | /api/photo-groups      | グループ登録           |
| PUT    | /api/photo-groups/{id} | グループ編集           |
| DELETE | /api/photo-groups/{id} | グループ削除           |

# 8. 補足

* Exif情報から撮影日・位置情報を自動取得
* 並び順はユーザーごとに保持
* ページング or 無限スクロール対応
* 写真の選択操作で編集・削除・一括削除可能
* UI上でチェックボックスを用いたマルチ選択対応
* マルチ選択による写真の一括登録対応（複数ファイルを同時にアップロード可能）
* マルチ選択による場所・説明の一括更新対応（場所・説明のチェックがONの項目のみ、選択した複数写真に同一の場所・説明を適用）
* マルチ選択による複数写真のダウンロード対応（ZIP圧縮でのダウンロード）
