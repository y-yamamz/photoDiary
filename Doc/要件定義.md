# PhotoDiary（アルバム管理）設計書

---

# 1. システム概要

撮影した写真を一元管理し、日記形式で閲覧できるアルバムアプリ。
ユーザーごとに写真・グループを管理し、日付・場所・カテゴリで整理・検索可能。

---

# 2. システム構成

* フロントエンド：React + TypeScript
* バックエンド：Spring Boot
* DB：MySQL
* ストレージ：ファイルサーバ or クラウド（S3想定）

---

# 3. 写真保存方式

## ■ 方針

### パターン①（推奨）

* 画像ファイル：サーバ or S3 に保存
* DB：パスのみ保持

```
例）/images/{userId}/{yyyy}/{MM}/{uuid}.jpg
```

### 理由

* DB肥大化防止
* パフォーマンス向上
* CDN利用可能

# 4. テーブル設計

| カラム名       | 型            | PK | NN | コメント        |
| ---------- | ------------ | -- | -- | ----------- |

| カラム名       | 型            | PK | NN | コメント     |

| group_id   | BIGINT       | ○  | ○  | グループID   |

# 6. API設計（概要）

| メソッド | URL         | 内容     |
| ---- | ----------- | ------ |
| POST | /api/login  | ログイン   |
| GET  | /api/photos | 写真一覧取得 |
| POST | /api/photos | 写真登録   |
| GET  | /api/groups | グループ一覧 |
| POST | /api/groups | グループ登録 |
