# PhotoDiary Docker デプロイ手順書
## 対象環境: Windows 11 + Docker Desktop（CIFS named volume 使用版）

---

## 概要

| コンポーネント | 技術 | 役割 |
|---|---|---|
| frontend | Nginx + React/Vite | UI配信・リバースプロキシ (ポート80) |
| backend | Tomcat + Spring Boot WAR | API・画像配信 (内部ポート8080) |
| MySQL | 既存サーバー (192.168.0.3) | データ永続化 |
| NAS | \\192.168.0.5\homedir\PhotoDiary | 写真ファイル保存 |

ブラウザからのすべてのリクエストは **Nginx (ポート80)** を経由します。
`/api/*` と `/images/*` は自動的にバックエンドコンテナへ転送されます。

```
ブラウザ → Nginx (port 80) ─┬─ 静的ファイル (React)
                             ├─ /api/**    → Tomcat (8080)
                             └─ /images/** → Tomcat (8080) → NAS (CIFS: \\192.168.0.5\homedir\PhotoDiary)
```

---

## 前提条件

- Windows 11
- Docker Desktop for Windows (WSL2 バックエンド)
- MySQL が `192.168.0.3:3306/photodb` で稼働中
- NAS `\\192.168.0.5\homedir` へネットワーク疎通があること

> **なぜ net use / mklink ではなく CIFS を使うのか？**
> Docker Desktop は WSL2（Hyper-V VM）上で動作するため、Windowsの `net use` マップドライブや
> `mklink` ジャンクションは WSL2 から参照できず、マウントしても中身が空になります。
> CIFS named volume は Docker が直接 NAS に接続するため、この制約を回避できます。

```powershell
# バージョン確認
docker --version
docker compose version
```

## デプロイ手順

### Step 1: リポジトリをローカルに配置

```powershell
# 任意のフォルダに配置（例）
cd D:\WORK
git clone <リポジトリURL> photoDiary
```

### Step 2: NAS認証情報を .env に設定

NASがゲストアクセス可能な場合は空のままでOKです。
認証が必要な場合は `.env` を編集します：

```env
NAS_USER=ユーザー名
NAS_PASSWORD=パスワード
```

NASへの疎通確認（Windowsから）：
```cmd
dir \\192.168.0.5\homedir\PhotoDiary
```
写真フォルダが表示されれば接続可能です。

> **net use / mklink を使わない理由**
> Docker Desktop は WSL2（Hyper-V VM）上で動くため、`net use` のマップドライブや
> `mklink` ジャンクションは WSL2 から中身が見えません（空ディレクトリになります）。
> CIFS named volume は Docker が直接 NAS に接続するためこの問題が発生しません。

### Step 3: Docker Desktop の File Sharing 設定は不要

CIFS named volume 方式では、Docker が直接 NAS に接続するため
Docker Desktop の **File Sharing 設定は変更不要**です。

### Step 4: 環境変数ファイルを作成

```powershell
cd D:\WORK\photoDiary\Doc\Deploy
copy .env.example .env
```

`.env` を編集（必要に応じて MySQL 接続情報を変更）：

```env
MYSQL_URL=jdbc:mysql://192.168.0.3:3306/photodb?useSSL=false&serverTimezone=Asia/Tokyo
MYSQL_USER=yama
MYSQL_PASSWORD=yama
FRONTEND_PORT=80
```

### Step 5: MySQLのテーブルを確認

MySQLが稼働中で `photodb` のテーブルが作成済みであることを確認します。
未作成の場合は `Doc/DB/` 配下のSQLファイルを順番に実行してください。

```powershell
# MySQL クライアントがある場合
mysql -h 192.168.0.3 -u yama -p photodb < Doc\DB\01_create_users.sql
mysql -h 192.168.0.3 -u yama -p photodb < Doc\DB\02_create_photo_groups.sql
mysql -h 192.168.0.3 -u yama -p photodb < Doc\DB\03_create_photos.sql
mysql -h 192.168.0.3 -u yama -p photodb < Doc\DB\04_create_tags.sql
mysql -h 192.168.0.3 -u yama -p photodb < Doc\DB\05_create_photo_tags.sql
```

### Step 6: コンテナをビルド・起動

プロジェクトルート（`D:\WORK\photoDiary`）で実行：

```powershell
docker compose -f Doc/Deploy/docker-compose.yml --env-file Doc/Deploy/.env up -d --build
```

初回ビルドは Gradle・npm のダウンロードがあるため **5〜15分** かかります。

### Step 7: 起動確認

```powershell
# コンテナ状態確認
docker compose -f Doc/Deploy/docker-compose.yml ps
```

```
NAME                   STATUS    PORTS
photodiary-frontend    running   0.0.0.0:80->80/tcp
photodiary-backend     running
```

```powershell
# NASマウント確認（開発環境の写真フォルダが見えればOK）
docker exec photodiary-backend ls /var/photodiary/images
```

### Step 8: ブラウザで動作確認

```
http://localhost/
```

## 設定変更

### MySQL接続先を変更する

`.env` の `MYSQL_URL` を変更して再起動します。

### NASの接続先を変更する

`docker-compose.yml` の `volumes` セクションの `device` を変更します：

```yaml
volumes:
  nas_photos:
    driver: local
    driver_opts:
      type: cifs
      device: //新しいNASのIP/共有名/PhotoDiary
      o: "addr=新しいNASのIP,username=${NAS_USER},password=${NAS_PASSWORD},file_mode=0777,dir_mode=0777,noperm"
```

変更後はボリュームを再作成します：
```powershell
docker compose -f Doc/Deploy/docker-compose.yml down -v
docker compose -f Doc/Deploy/docker-compose.yml up -d
```

### フロントエンドのポートを変更する

`.env` を変更します：
```env
FRONTEND_PORT=8080
```

### JWTシークレットキーを変更する（本番推奨）

`docker-compose.yml` の environment に追加します：
```yaml
environment:
  - JWT_SECRET=<32文字以上のランダム文字列>
```

## トラブルシューティング

### ペー��が表示されない (502 Bad Gateway)

Tomcat の起動完了まで 30〜60秒かかります。ログを確認してください：

```powershell
docker logs photodiary-backend -f
```

`Server startup in [XXXX] milliseconds` が表示されれば起動完了です。

### 写真が表示されない

**① NASマウントの確認**

```powershell
# コンテナ内に写真フォルダが見えるか確認
docker exec photodiary-backend ls /var/photodiary/images
```

何も表示されない場合は以下を確認：

- NAS への疎通があるか（Windowsから確認）
  ```cmd
  dir \\192.168.0.5\homedir\PhotoDiary
  ```
- `.env` の `NAS_USER` / `NAS_PASSWORD` が正しいか
- ボリュームを再作成したか
  ```powershell
  docker compose -f Doc/Deploy/docker-compose.yml down -v
  docker compose -f Doc/Deploy/docker-compose.yml up -d
  ```

**② 書き込み権限の確認**

```powershell
docker exec photodiary-backend touch /var/photodiary/images/.test
```

エラーが出る場合は NAS 側の書き込み権限を確認してください。

### MySQLに接続できない

```powershell
docker exec photodiary-backend bash -c "cat /proc/net/tcp" | head
```

- `.env` の `MYSQL_URL`, `MYSQL_USER`, `MYSQL_PASSWORD` が正しいか確認
- MySQL サーバーが稼働しているか確認
- MySQL ユーザーに Docker ホスト IP からのアクセス権があるか確認

```sql
-- MySQL側で権限付与（必要な場合）
GRANT ALL PRIVILEGES ON photodb.* TO 'yama'@'%' IDENTIFIED BY 'yama';
FLUSH PRIVILEGES;
```

### ビルドエラー (Gradle)

```powershell
docker compose -f Doc/Deploy/docker-compose.yml build --no-cache --progress=plain backend 2>&1 | Select-Object -Last 50
```

よくある原因：
- `backend/gradle/wrapper/gradle-wrapper.jar` がリポジトリに含まれていない
- インターネット接続がない（Maven Central へのアクセスが必要）

### ポート80が使用中

`.env` でポートを変更します：
```env
FRONTEND_PORT=8080
```

ポートを使用しているプロセスを確認するには：
```powershell
netstat -ano | findstr :80
```
